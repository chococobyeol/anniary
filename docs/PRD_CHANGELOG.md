## [2026-03-17] 백로그 UX 및 태그 (v3.2 구현 반영)

### 변경 대상

- `prd_v2.md` §6.4 좌측 패널, §11 날짜 없는 작업
- `prd_v2_ux.md` §7 backlog 모드
- `prd_v2_entity.md` 태그 관련 (item.tags 반영)

### 1. 좌측 패널 — backlog 역할 (prd_v2.md §6.4)

| 구분 | 내용 |
|------|------|
| **수정 전** | "backlog"만 나열. 입력 방식·태그·표시 제한에 대한 기술 없음. |
| **수정 후** | backlog는 **인박스**: textarea + 추가 버튼으로 빠른 생성, **태그** 선택(기본 "일반"), **태그별 그룹** 표시, **표시 개수** 설정(전부 보기 / 최근 N개, updatedAt 기준). 날짜·기간·메모는 디테일에서 할당. |

### 2. 날짜 없는 작업 — 기대 동작 (prd_v2.md §11.4)

| 구분 | 내용 |
|------|------|
| **수정 전** | 날짜 없이 생성 가능, 나중에 날짜 할당, 완료 처리, 날짜로 이동, 메모로만, range로 할당. |
| **수정 후** | 위에 더해 **태그 부여 가능**(생성 시 선택 또는 "일반"), **태그별로 목록 그룹화**, **표시 개수 제한 옵션**(설정에서 최근 N개만 보기). |

### 3. backlog 모드 상세 (prd_v2_ux.md §7)

| 구분 | 내용 |
|------|------|
| **수정 전** | 목적: 날짜 없는 item 관리, 빠른 생성/편집. 포함: no-date task/note, range 미할당, date 미할당. 기능: create, edit title, set status, assign date/range, open detail, drag to board. 정렬: pinned → in-progress → updated desc. |
| **수정 후** | **입력**: 상단에 textarea + 추가 버튼만. Enter=추가, Shift+Enter=줄바꿈. **태그**: 입력 영역 아래 태그 칩(기존 태그 + "일반") 및 "+ 새 태그" 입력. 선택 없으면 "일반". **목록**: 태그별 그룹 표시(그룹 헤더에 태그명·개수). **표시 제한**: 설정에서 "전부 보기" 또는 "최근 N개"(updatedAt 기준). **정렬**: 전체를 updatedAt 내림차순 후 N개 제한, 그 안에서 태그별 그룹. 완료 항목은 접이식 "완료됨" 섹션. (날짜/기간/메모는 백로그 입력에서 제거, 디테일·보드에서 할당.) |

### 4. 데이터 모델 — 태그 (prd_v2_entity.md)

| 구분 | 내용 |
|------|------|
| **수정 전** | "tag 엔티티"를 초기에는 빼는 것 목록에 포함. |
| **수정 후** | **태그는 별도 tag 엔티티 없이** Item에 `tags?: string[]` 필드로 보관. 빈 값/미지정 시 UI에서는 "일반"으로 표시. 필터·검색은 이 문자열 배열 기준. |

### 사유

- 백로그를 mwohaji처럼 **최소 입력(텍스트 + 태그)** 으로 빠르게 쌓고, 날짜/기간은 나중에 디테일·보드에서 할당.
- kind(task/note/event) 선택을 백로그 입력에서 제거해 진입 장벽 축소.
- 태그로 분류·필터·그룹 보기 제공.
- 항목이 많을 때 "최근 N개"로 목록 제한 가능하도록 설정 추가.
