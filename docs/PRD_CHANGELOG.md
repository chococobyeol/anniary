## [2026-03-24] 필터·분류 축 — 태그 기준 (kind 비전제)

### 변경 대상

- `prd_v2_ux.md` §9 filter 모드
- `prd_v2.md` §14.2 필터, §17.4 날짜 원칙 한 줄
- `prd_v2_entity.md` §4.2 item, §6, §8, §19
- `IMPLEMENTATION_CHECKLIST.md` — Filter 패널 한 줄 설명

### 1. filter 모드 예시 (prd_v2_ux.md §9)

| 구분 | 내용 |
|------|------|
| **수정 전** | 필터 예: task only, note only, event only 등 **Item.kind 축**이 전제된 예시. |
| **수정 후** | **분류 기준** 문단 추가: 사용자 구분·UI는 `item.tags` 중심, `item.kind`는 스키마/기본값용이며 필터 1차 축 아님. 필터 예시를 **특정 태그만**, 태그 다중 선택, 기본 "일반" 등으로 교체. in-progress / hide done / overlay / range / memo 예시는 유지. |

### 2. 본문 필터 절 (prd_v2.md §14.2)

| 구분 | 내용 |
|------|------|
| **수정 전** | 예상 필터 bullet 목록만 (이미 "특정 태그만" 포함). kind와의 관계 명시 없음. |
| **수정 후** | **분류 축** 단락 추가: 필터·검색은 `item.tags` 기준, kind 기반 "종류만 보기" 필터는 전제하지 않음. |

### 2b. 날짜 원칙 (prd_v2.md §17.4)

| 구분 | 내용 |
|------|------|
| **수정 전** | `task/note/backlog는 날짜가 optional` |
| **수정 후** | `item(백로그 포함)` 으로 통일하고 태그·kind 스키마는 §14.2·엔티티 문서 참조로 연결. |

### 3. 엔티티 문서 (prd_v2_entity.md)

| 구분 | 내용 |
|------|------|
| **수정 전** | item을 task/note/event로 나누지 않고 **`item.kind`로 구분**한다는 서술. §8에서 kind로 no-date task/note 구분 예시. §19에서 "`item.kind`로 구분" 요약. |
| **수정 후** | §4.2: **tags가 사용자 구분·필터·검색 1차 축**, kind는 스키마 유지·기본 task·UI에서 종류 미선택. §6: 한 item 타입 유지 이유는 동일하되 kind/tags 역할 분리 명시. §8: 제목·내용을 **no-date item** 중심으로 정리, kind로 UI 분리하지 않음. §19 핵심 판단 2번을 **tags = 사용자 분류·필터 축**으로 수정. |

### 4. 구현 체크리스트

| 구분 | 내용 |
|------|------|
| **수정 전** | `Filter 패널 — view filter` |
| **수정 후** | 동일 항목에 괄호로 **태그·상태·메모 등, Item.kind 기반 필터 아님** 명시. |

### 사유

- 구현(v3.2)과 일치: 백로그·디테일은 **태그** 중심이며 신규 item은 **kind 고정(task)** 에 가깝다.
- PRD에 남아 있던 **task/note/event 필터 예시**는 현재 제품 방향과 어긋나므로 문서만 정합성 맞춤.

## [2026-03-17] 백로그 UX 및 태그 (v3.2 구현 반영)

### 변경 대상

- `prd_v2.md` §6.4 좌측 패널, §11 날짜 없는 작업
- `prd_v2_ux.md` §7 backlog 모드
- `prd_v2_entity.md` 태그 관련 (item.tags 반영)

### 1. 좌측 패널 — backlog 역할 (prd_v2.md §6.4)

| 구분 | 내용 |
|------|------|
| **수정 전** | "backlog"만 나열. 입력 방식·태그·표시 제한에 대한 기술 없음. |
| **수정 후** | backlog는 **인박스**: textarea + 추가 버튼으로 빠른 생성, **태그** 선택(기본 "일반"), **태그별 그룹** 표시, **표시 개수** 설정(전부 보기 / 최근 N개, updatedAt 기준). 날짜·기간·메모는 디테일에서 할당. |

### 2. 날짜 없는 작업 — 기대 동작 (prd_v2.md §11.4)

| 구분 | 내용 |
|------|------|
| **수정 전** | 날짜 없이 생성 가능, 나중에 날짜 할당, 완료 처리, 날짜로 이동, 메모로만, range로 할당. |
| **수정 후** | 위에 더해 **태그 부여 가능**(생성 시 선택 또는 "일반"), **태그별로 목록 그룹화**, **표시 개수 제한 옵션**(설정에서 최근 N개만 보기). |

### 3. backlog 모드 상세 (prd_v2_ux.md §7)

| 구분 | 내용 |
|------|------|
| **수정 전** | 목적: 날짜 없는 item 관리, 빠른 생성/편집. 포함: no-date task/note, range 미할당, date 미할당. 기능: create, edit title, set status, assign date/range, open detail, drag to board. 정렬: pinned → in-progress → updated desc. |
| **수정 후** | **입력**: 상단에 textarea + 추가 버튼만. Enter=추가, Shift+Enter=줄바꿈. **태그**: 입력 영역 아래 태그 칩(기존 태그 + "일반") 및 "+ 새 태그" 입력. 선택 없으면 "일반". **목록**: 태그별 그룹 표시(그룹 헤더에 태그명·개수). **표시 제한**: 설정에서 "전부 보기" 또는 "최근 N개"(updatedAt 기준). **정렬**: 전체를 updatedAt 내림차순 후 N개 제한, 그 안에서 태그별 그룹. 완료 항목은 접이식 "완료됨" 섹션. (날짜/기간/메모는 백로그 입력에서 제거, 디테일·보드에서 할당.) |

### 4. 데이터 모델 — 태그 (prd_v2_entity.md)

| 구분 | 내용 |
|------|------|
| **수정 전** | "tag 엔티티"를 초기에는 빼는 것 목록에 포함. |
| **수정 후** | **태그는 별도 tag 엔티티 없이** Item에 `tags?: string[]` 필드로 보관. 빈 값/미지정 시 UI에서는 "일반"으로 표시. 필터·검색은 이 문자열 배열 기준. |

### 사유

- 백로그를 mwohaji처럼 **최소 입력(텍스트 + 태그)** 으로 빠르게 쌓고, 날짜/기간은 나중에 디테일·보드에서 할당.
- kind(task/note/event) 선택을 백로그 입력에서 제거해 진입 장벽 축소.
- 태그로 분류·필터·그룹 보기 제공.
- 항목이 많을 때 "최근 N개"로 목록 제한 가능하도록 설정 추가.
